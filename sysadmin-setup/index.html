    Setup

    Этот документ описывает конфигурирование сервисов с точки зрения системного администратора.

    Processing Service

    Processing Service - (aka CPService) "сердце" проекта, реализует B2B API и занимается процессингом ордеров по обмену валют.

    Системному администратору предоставляется три интерфейса управления:

    1. System Administrator Tool - (UNDER DEVELOPMENT) Интерфейс командной строки. Утилита доступная в терминале контейнера processing.
    2. Configuration Approach - временные конфигурационные ключи setup.xxxxxx которые можно предварительно добавить к конфигурации сервиса (напр. в Kubernetes ConfigMaps/Secrets или локальные конфигурационные файлы разработчика)
    3. Low Level - запросы через curl на localhost/unix socket. Данный вариант может опережать возможности System Administrator Tool и Configuration Approach

    В основе заложена идея, что любую конфигурацию можно сделать одинаково, вне зависимости от выбранного способа. Исключением является лишь то, что конфигурирование через ключи выполняется только на старте сервиса (требуется рестарт).

    System Administrator Tool

    Это утилита командной строки которая поставляется вместе с Processing Service. Используется внутри контейнера с Processing Service.

    cpadmin --help

    Low Level

    Это утилита командной строки которая поставляется вместе с Processing Service. Используется внутри контейнера с Processing Service.

    curl --request GET/POST/PUT/DELETE ... http://localhost:8081/v2/sysadmin/XXXXXXXXXXXXXXXX

    Configuration Approach

    Этот способ подразумевает добавление ключей setup.xxxxxx в конфигурацию сервиса. В Kubernetes для этого предусмотрены следующие объекты:

    • processing-envvars-setup - ConfigMap
    • processing-setup - Secrets

    Конфигурационные ключи добавляются сюда на временной основе. После успешного запуска сервиса, эти временные ключи можно удалить (или оставить на память, с небольшим риском, что они применятся когда-то в будущем)

    Configuration Keys Merging

    Конфигурация сервиса построена на наборе ключ-значение. При старте, сервис производит мержинг всех конфигурационных источников, таких как переменные окружения, Secrets-ы, ConfigMap-ы, дефолтные конфиги из контейнера. Порядок мержинга задается параметрами запуска сервиса.

    По-сути всю конфигурацию можно выполнить через переменные окружения, без единого конфигурационного файла.

    Доступные настройки

    Список мерчантов

    # this will write list of merchants to logger at startup.
setup.merchantListOnStart = true

    $ curl --request GET http://localhost:8081/v2/sysadmin/merchant

    Создание мерчанта

    setup.merchantCreate.DEVTEAM.parent = INTERMEDIATE
setup.merchantCreate.DEVTEAM.description = Merchant Dev Team (sandbox)
setup.merchantCreate.DEVTEAM.enabled = true

setup.merchantCreate.DEVTEAM__EMU.parent = DEVTEAM
setup.merchantCreate.DEVTEAM__EMU.description = Sub Merchant Dev Team (sandbox)
setup.merchantCreate.DEVTEAM__EMU.enabled = true

setup.merchantCreates = DEVTEAM DEVTEAM__EMU

    $ curl --request POST --data '{"name":"DEVTEAM","description":"Merchant Dev Team","parent":"INTERMEDIATE","enabled":true}' http://localhost:8081/v2/sysadmin/merchant

    где,

    • DEVTEAM - Имя создаваемого мерчанта
    • INTERMEDIATE - Имя родительского мерчанта
    • Merchant Dev Team - Описание мерчанта (отображается в дашборде)
    • true - Мерчант включен (разрешена работа). Может быть false.

    Создание API ключей

    setup.accessKeyCreate.53f75b533edfe38d176f02c270dac6c0.merchantName = DEVTEAM
setup.accessKeyCreate.53f75b533edfe38d176f02c270dac6c0.passphrase = meow-meow
setup.accessKeyCreate.53f75b533edfe38d176f02c270dac6c0.secret = xxxxxxxxxxxxxxxxxxx

setup.accessKeyCreate.602994ff31e5c23f17bdfe894e14b730.merchantName = DEVTEAM__EMU
setup.accessKeyCreate.602994ff31e5c23f17bdfe894e14b730.passphrase = bow-wow
setup.accessKeyCreate.602994ff31e5c23f17bdfe894e14b730.secret = yyyyyyyyyyyyyyyyyyy

setup.accessKeyCreates = 53f75b533edfe38d176f02c270dac6c0 602994ff31e5c23f17bdfe894e14b730

    $ curl --request POST --data '{"name":"DEVTEAM","key":"602994ff31e5c23f17bdfe894e14b730","passphrase":"bow-wow","secret":"xxxxxxxxxxxxxxxxxxx"}' http://localhost:8081/v2/sysadmin/access-key
$ curl --request POST --data '{"name":"DEVTEAM","key":"53f75b533edfe38d176f02c270dac6c0","passphrase":"meow-meow","secret":"yyyyyyyyyyyyyyyyyyy"}' http://localhost:8081/v2/sysadmin/access-key

    где,

    • DEVTEAM - Имя создаваемого мерчанта
    • 602994ff31e5c23f17bdfe894e14b730, 602994ff31e5c23f17bdfe894e14b730 - API Key
    • bow-wow, meow-meow - API Passphrase
    • xxxxxxxxxxxxxxxxxxx, yyyyyyyyyyyyyyyyyyy - API Secret

    Установка Merchant Pair

    setup.merchantPairSet.pair1.merchant = INTERMEDIATE
setup.merchantPairSet.pair1.marketCurrency = USD
setup.merchantPairSet.pair1.tradeCurrency = BTC
setup.merchantPairSet.pair1.use = false

setup.merchantPairSet.pair2.merchant = INTERMEDIATE
setup.merchantPairSet.pair2.marketCurrency = EUR
setup.merchantPairSet.pair2.tradeCurrency = BTC
setup.merchantPairSet.pair2.use = null

setup.merchantPairSet.pair3.merchant = INTERMEDIATE
setup.merchantPairSet.pair3.marketCurrency = USD
setup.merchantPairSet.pair3.tradeCurrency = ETH
setup.merchantPairSet.pair3.use = true

setup.merchantValueSets = pair1 pair2 pair3

    $ curl --request POST --data '{"market_currency":"USD","trade_currency":"BTC","use":false}' http://localhost:8081/v2/sysadmin/merchant/INTERMEDIATE/pair
$ curl --request POST --data '{"market_currency":"EUR","trade_currency":"BTC","use":null}' http://localhost:8081/v2/sysadmin/merchant/INTERMEDIATE/pair
$ curl --request POST --data '{"market_currency":"USD","trade_currency":"ETH","use":true}' http://localhost:8081/v2/sysadmin/merchant/INTERMEDIATE/pair

    где,

    • INTERMEDIATE - Имя мерчанта которому будет установлена торговая пара
    • USD, EUR - Валюта рынка
    • ETH, BTC - Валюта товара
    • true, null, false - значение флага use

    Список Merchant Values

    # this will write list of Merchant Values to logger at startup for INTERMEDIATE and DEVTEAM merchants.
setup.merchantValueListOnStartForMerchants = INTERMEDIATE DEVTEAM

    $ curl --request GET http://localhost:8081/v2/sysadmin/merchant/INTERMEDIATE/value
$ curl --request GET http://localhost:8081/v2/sysadmin/merchant/DEVTEAM/value

    где,

    • INTERMEDIATE, DEVTEAM - Имя мерчанта по которому нужен список Merchant Values

    Установка Merchant Value

    setup.merchantValueSet.val001.merchant = INTERMEDIATE
setup.merchantValueSet.val001.marketCurrency = USD
setup.merchantValueSet.val001.tradeCurrency = BTC
setup.merchantValueSet.val001.key = CRYPTO_SELL.DEFAULT.LIMIT.MIN_FIAT_AMOUNT
setup.merchantValueSet.val001.value = 0.0001

setup.merchantValueSet.val002.merchant = INTERMEDIATE
setup.merchantValueSet.val002.marketCurrency = USD
setup.merchantValueSet.val002.tradeCurrency = BTC
setup.merchantValueSet.val002.key = CRYPTO_SELL.DEFAULT.LIMIT.MAX_FIAT_AMOUNT
setup.merchantValueSet.val002.value = 9.99999999

setup.merchantValueSet.val003.merchant = INTERMEDIATE
setup.merchantValueSet.val003.marketCurrency = USD
setup.merchantValueSet.val003.tradeCurrency = ATOM
setup.merchantValueSet.val003.key = CRYPTO_SELL.DEFAULT.LIMIT.MAX_FIAT_AMOUNT
setup.merchantValueSet.val003.value = null

setup.merchantValueSets = val001 val002 val003

    $ curl --request PUT --data '"0.0001"' http://localhost:8081/v2/sysadmin/merchant/INTERMEDIATE/BTC/USD/CRYPTO_SELL.DEFAULT.LIMIT.MIN_FIAT_AMOUNT
$ curl --request PUT --data '"9.99999999"' http://localhost:8081/v2/sysadmin/merchant/INTERMEDIATE/value/BTC/USD/CRYPTO_SELL.DEFAULT.LIMIT.MAX_FIAT_AMOUNT
$ curl --request PUT --data 'null' http://localhost:8081/v2/sysadmin/merchant/INTERMEDIATE/ATOM/USD/CRYPTO_SELL.DEFAULT.LIMIT.MAX_FIAT_AMOUNT

    где,

    • INTERMEDIATE - Имя мерчанта которому будет установлено значение
    • CRYPTO_SELL.DEFAULT.LIMIT.MAX_FIAT_AMOUNT, CRYPTO_SELL.DEFAULT.LIMIT.MAX_FIAT_AMOUNT - Имена Merchant Value
    • 0.0001, 9.99999999, null - значения

    Удаление Merchant Value

    setup.merchantValueRemove.val001.merchant = INTERMEDIATE
setup.merchantValueRemove.val001.marketCurrency = USD
setup.merchantValueRemove.val001.tradeCurrency = BTC
setup.merchantValueRemove.val001.key = CRYPTO_SELL.DEFAULT.LIMIT.MIN_FIAT_AMOUNT

setup.merchantValueRemove.val002.merchant = INTERMEDIATE
setup.merchantValueRemove.val001.marketCurrency = USD
setup.merchantValueRemove.val001.tradeCurrency = BTC
setup.merchantValueRemove.val002.key = CRYPTO_SELL.DEFAULT.LIMIT.MAX_FIAT_AMOUNT

setup.merchantValueRemoves = val001 val002

    $ curl --request DELETE http://localhost:8081/v2/sysadmin/merchant/INTERMEDIATE/value/BTC/USD/CRYPTO_SELL.DEFAULT.LIMIT.MIN_FIAT_AMOUNT
$ curl --request DELETE http://localhost:8081/v2/sysadmin/merchant/INTERMEDIATE/value/BTC/USD/CRYPTO_SELL.DEFAULT.LIMIT.MAX_FIAT_AMOUNT

    где,

    • INTERMEDIATE - Имя мерчанта которому будет удалено значение
    • CRYPTO_SELL.DEFAULT.LIMIT.MIN_FIAT_AMOUNT, CRYPTO_SELL.DEFAULT.LIMIT.MAX_FIAT_AMOUNT - Имена Merchant Value которые нужно удалить

    Установка HOT-адреса для мерчанта

    setup.setHotAddress.addr1.cryptoWallet = presentation1
setup.setHotAddress.addr1.name = cryptopay-swagger-ada-hot
setup.setHotAddress.addr1.currency = ADA
setup.setHotAddress.addr1.address = 2cWKMJemoBajB4bxUnQnGveCN3BWpoFeVKt7UgBAY2kypfSHuRTHmxb87mdZqhxnY1etS
setup.setHotAddress.addr1.merchant = SWAGGER

setup.setHotAddress.addr2.cryptoWallet = presentation1
setup.setHotAddress.addr2.name = cryptopay-swagger-trx-hot
setup.setHotAddress.addr2.currency = TRX
setup.setHotAddress.addr2.address = TVmK1i9qhPgVzTx5NzDNGfr61fymQcofvm
setup.setHotAddress.addr2.merchant = SWAGGER

setup.setHotAddresses = addr1 addr2

    $ curl --request PUT --data '{"cryptoWallet":"presentation1","name":"cryptopay-swagger-ada-hot","address":"2cWKMJemoBajB4bxUnQnGveCN3BWpoFeVKt7UgBAY2kypfSHuRTHmxb87mdZqhxnY1etS"}' http://localhost:8081/v2/sysadmin/merchant/SWAGGER/hot-crypto-address/ADA
$ curl --request PUT --data '{"cryptoWallet":"presentation1","name":"cryptopay-swagger-trx-hot","address":"TVmK1i9qhPgVzTx5NzDNGfr61fymQcofvm"}' http://localhost:8081/v2/sysadmin/merchant/SWAGGER/hot-crypto-address/TRX

    где,

    • presentation1 - Имя крипто кошелька (задается при создании кошелька), который обслуживает даный адрес
    • cryptopay-swagger-ada-hot, cryptopay-swagger-trx-hot - Имя адреса (поле name в pss-provider-wtf2)
    • ADA, TRX - Код криптовалюты
    • 2cWKMJemoBajB4bxUnQnGveCN3BWpoFeVKt7UgBAY2kypfSHuRTHmxb87mdZqhxnY1etS, TVmK1i9qhPgVzTx5NzDNGfr61fymQcofvm - Адреса блокчейна
    • SWAGGER - Имя мерчанта которому предназначан HOT адрес
