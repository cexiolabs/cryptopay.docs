    Merchant

    Family

    Merchant Family - это родительские отношения между мерчантами. Технически у любого мерчанта есть родитель, кроме технического мерчанта ROOT который служит вершиной дерева. Merchant Family позволяет реализовать наследование Pairs, Values, ExchangeCompoments, CrytoWalletComponents, etc.

    Например есть мерчант Ecommpay. Все дети этого мерчанта будут использовать его Pairs, Values, ExchangeCompoments, CrytoWalletComponents, если не имеют личных настроек. Таким образом количество конфигурационных записей сводится к минимуму.

    Использование

    • Получить список родителей

      SELECT "tree_merchant_id"
FROM "cp2_merchant"."vw_merchant_family"
WHERE "merchant_id" = 7 AND "depth" > 0
ORDER BY "depth" DESC

    • Получить список детей

      SELECT "merchant_id"
FROM "cp2_merchant"."vw_merchant_family"
WHERE "tree_merchant_id" = 2 AND "depth" > 0
ORDER BY "depth" DESC

    • Получить ближайшего родителя

      SELECT "tree_merchant_id"
FROM "cp2_merchant"."vw_merchant_family"
WHERE "merchant_id" = 7 AND "depth" = 1

    • Получить промежуточных мерчантов до требуемого родителя

      SELECT unnest("path")
FROM "cp2_merchant"."vw_merchant_family"
WHERE "merchant_id" = 7 AND "tree_merchant_id" = 2

    Объекты базы данных

    merchant

    Таблица идентификатор мерчанта

    vw_merchant_family

    Warning

    Вносите изменения в работу этого вида с большой осторожностью! Заэффектит все :)

    Вид который разворачивает родительские отношения по всем мерчантам, со следующим списком колонок:

    • merchant_id - идентификатор мерчанта
    • tree_merchant_id - идентификатор мерчанта вверх по дереву (включая самого себя, это важное свойство!)
    • depth - глубина связи (сколько шагов в родительской цепочке)
    • path - массив, путь к мерчанту родителю (какие мерчанты стоят на пути)

    Благодаря этому виду, можно в один запрос узнать всех родителей или всех детей по интересующему мерчанту. Также этот вид является основой для работы механизма переопределения Pairs и Values.

    Pairs

    Merchant Pairs - это набор торговых пар доступных для мерчанта. Реализовано в виде отдельной таблицы, для обеспечения FOREIGN KEY в ордерах процессинга и в трейдах обменника.

    Значения наследуются от мерчанта-родителя к мерчантам-детям с возможностью переопределения, что дает возможность не плодить одинаковые значения для каждого мерчанта.

    • При чтении торговых пар: Выполняется waterfall от текущего мерчанта до корневого в поиске торговых пар. Другими словами если у мерчанта нет торговой пары, берем торговую пару родителя (рекурсивно до корневого мерчанта)
    • При записи торговой пары: Пара создается/удаляется конкретному мерчанту, таким образом выполняется переопределение значения родителя. На вход передается идентификатор мерчанта, пара валют и use флаг:
      • use is TRUE: Создание пары (если пары еще нет). Включение пары (разрешение на использование). Разрешает пару для мерчанта и всех его потомков.
      • use is FALSE: Создание пары (если пары еще нет). Выключение пары (запрет на использование). Запрещает пару для мерчанта, HO разрешает включить эту пару детям. Это способ объявить допустимые пары у родителя, а конкретные дети будут их включать их себе лично.
      • use is NULL: Создание пары (если пары еще нет). Но игнорирование(пропуск) при чтении. Используется как возможность вернутся к настройкам мерчанта родителя (запись из таблицы не может быть удалена, т. к. на строке есть связи на ордера)

    Объекты базы данных

    pair

    Таблица значений ключей, включают в себя:

    • merchant_id - ссылка на мерчанта
    • market_currency - Ссылка на справочник валют. Валюта рынка (которая справа в symbol XXX/XXX).
    • trade_currency - Ссылка на справочник валют. Валюта товара (которая слева в symbol XXX/XXX).
    • deny - Boolean, Nullable. Поле через которое реализуется поведение включения/выключения/игнорирования.

    Readonly

    Модификация этой таблицы доступна только через sp_pair_set

    UPDATE Restrict

    Разрешено обновление ТОЛЬКО поля deny

    pair_hist

    Таблица истории изменений поля deny таблицы pair (заполняется тригером при модификации таблицы pair), включают в себя:

    • pair_id - ссылка на торговую пару
    • old_deny - предыдущее значение ключа
    • new_deny - новое значение ключа
    • utc_created_at - дата модификации

    vw_pair

    Вид который разворачивает торговые пары по каждому мерчанту. Выполняет работу по наследованию с переопределением.

    sp_pair_set

    Процедура(функция) для установки торговой пары. На вход передается идентификатор мерчанта, пара валют и use флаг.

    Values

    Merchant Values - это набор финансовых чисел хранящихся в виде ключ-значение, на основе справочника ключей. Эти числа позволяют создавать наследуемые конфигурации лимитов, комиссий, etc.

    Значения наследуются от мерчанта-родителя к мерчантам-детям с возможностью переопределения, что дает возможность не плодить одинаковые значения для каждого мерчанта.

    • При чтении значения ключа: Выполняется waterfall от текущего мерчанта до корневого в поиске значения. Другими словами если у мерчанта нет установленного ключа, берем значение родителя (рекурсивно до корневого мерчанта)
    • При записи значения ключа: Значение устанавливается конкретному мерчанту, таким образом выполняется переопределение значения родителя. Установка NULL означает что, конкретный мерчант прерывает наследование родительских значений для своих детей (обязывает детей установить свои значения).
    • При удалении значения ключа: Удаление ключа у конкретного мерчанта означает что, конкретный мерчант снова начинает использовать значение ключа от родителя.

    Ограничения

    В справочнике ключей указываются правила наследования, которые создают ограничения на переопределяемые значения. Например в справочнике ключей правило MIN_DEPOSIT_AMOUNT.BTC = ge, мерчант-родитель имеет установленное значение MIN_DEPOSIT_AMOUNT.BTC = 0.001, эта конфигурация разрешает мерчанту-ребенку устанавливать ТОЛЬКО значения выше/равно 0.001. Ограничения реализуются на уровне базы данных.

    Использование

    Аpplication уровень в коде формирует свои объекты настроек на базе имен ключей, который может меняться от релиза к релизу.

    Не забываем обратную совместимость

    При изменении формата ключей заботьтесь об обратной совместимости. Несколько версий сервиса может работать в базой данных одновременно. Новый релиз не должен нарушать работу запущенных сервисов.

    Объекты базы данных

    value_dict

    Таблица справочник ключей, включают в себя:

    • name - уникальное имя ключа
    • override_rule - правило наследования (именование взято из Bash)
      • gt - is greater than
      • ge - is greater than or equal to
      • le - is less than or equal to
      • lt - is less than

    value

    Таблица значений ключей, включают в себя:

    • merchant_id - ссылка на мерчанта
    • name - ссылка на справочник ключей
    • value - значение ключа (nullable)

    Readonly

    Модификация этой таблицы доступна только через sp_value_set

    value_hist

    Таблица истории изменений значений ключей (заполняется тригером при модификации таблицы value), включают в себя:

    • merchant_id - ссылка на мерчанта
    • name - ссылка на справочник ключей
    • old_value - предыдущее значение ключа
    • new_value - новое значение ключа
    • utc_created_at - дата модификации значения

    vm_value

    Вид который разворачивает значения по каждому мерчанту. Выполняет работу по наследованию с переопределением.

    sp_value_set

    Процедура(функция) для установки значения.

    sp_value_remove

    Процедура(функция) для удаления значения.
