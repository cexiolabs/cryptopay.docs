    Database

    Проект /cryptopay/database содержит шаблонизированные исходники PostgreSQL базы данных проекта Cryptopay.

    Правила

    • Избегать использование UPDATE (INSERT as much as possible)
    • Избегать дубликатов данных (связь через идентификаторы)

    Структура проекта

    .
├── bin
│   └── build.js                          Скрипт для компиляции SQL шаблонов

    Компиляция

    Исходные updates скрипты используют шаблонизатор Mustache. Шаблонизатор необходим для подстановки имен пользователей, имен схем, имен tablespace, etc. Используя синтаксис Mustache + значения их конфигурационных файлов database*.config на выходе компиляции получаем чистый SQL с нужными подстановками. [!] Mustache поддерживает итерацию, что позволяет генерировать много INSERT-ов через несколько строк шаблона.

    Компиляция скриптов производится с помощью ./bin/build.js.

    В контексте для Mustache используется мержинг database.config + файла переданого в --envConfigurationFile + переменных окружения.

    Компиляция всех версий:

    Деплой

    Технически, скомпилированная версия это чистый SQL который может быть выполненен руками DBA.

    Для автоматизации развертывания версий используется MigrationManager (из @cexiolabs/cryptopay.sql), который позволяет выполнять install/rollback до нужной версии базы.

    Консольная версия MigrationManager доступна через докер-образ (который является базовым образом для сборки релиза).

    Note: --env TARGET_VERSION="vXX.XX" для ограничения версии

    Install

    $ docker run --rm --tty --interactive --volume ${PWD}/.dist:/var/local/cexiolabs/migration --env POSTGRES_URL="postgres://postgres@host.docker.internal:5432/cryptopay-devel" --env TARGET_VERSION="v02.01" cexiolabs/migration-postgres:latest install

    Релизный образ

    Релиз версии базы данных поставляется в виде докер-контейнера. Релизный контейнер выкладывается в продуктивный докер репозиторий. Запуск релизного контейнера выполняется с помощью Kubertenes Job (см. devops pipelines).

    Сборка релизного контейнера

    Пример сборки релизного контейнера который включает в себя версии из диапазона v02.00 - v02.05

    docker build --tag devdocker.infra.kube/cryptopay/database-devel --file docker/Dockerfile --build-arg ENV=devel --build-arg VERSION_FROM=v02.00 --build-arg VERSION_TO=v02.05 .

    Использование релизного контейнера

    Опционально: --env TARGET_VERSION="vXX.XX"

    Install

    docker run --rm --tty --interactive --env POSTGRES_URL="postgres://postgres@host.docker.internal:5432/cryptopay-devel" devdocker.infra.kube/cryptopay/database-devel install

    Rollback

    docker run --rm --tty --interactive --env POSTGRES_URL="postgres://postgres@host.docker.internal:5432/cryptopay-devel" devdocker.infra.kube/cryptopay/database-devel rollback
