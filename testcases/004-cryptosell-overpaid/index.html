    004 Crypto Sell With Overpaid

    Description

    Cryptocurrency payment (sale) scenario with overpaid by order amount

    Preconditions

    • Check your API Key related to TARGET Merchant
    • Check your are use a correct environment
    • Prepare JSON body for Create order request

    • In JSON body, in block "from" add optional field "accountId", if need to create an order based on cryptocurrency, based on a REJECTED order

    • Grab info from service settings:

      • How many confirmations need to process an order on choosen currency
    • Configure TARGET Merchant configuration like a limits, commissions, order timeouts, etc.
    • Subscribe HTTP callbacks for the topic DEPOSIT, ORDER

    Steps

    1. Create a CRYPTO_SELL order by call API Create order
      • HTTP 201
      • proper body content
      • check NEW order notification in ORDER topic
        • NEW order notification received
        • proper body content
      • check AWAIT_DEPOSIT order notification in ORDER topic
        • AWAIT_DEPOSIT order notification received
        • proper body content
    2. Grab from.amount and deposit.address. Make crypto deposit with larger amount than from.amount. Wait for 0-confirmation (transaction in memory pool).
      • Check for second notification in ORDER topic
        • REJECTED order notification received
        • status = "REJECTED"
        • rejectReason = "Overpaid. The order was rejected due received large amount."
        • state = "AWAIT_DEPOSIT"
        • paidStatus = "OVER_PAID"
        • deposit.transactions includes info of blockchain transaction

    Postconditions

    1. Return cryptocurrency to the user through withdrawal from the account - Make withdrawal
