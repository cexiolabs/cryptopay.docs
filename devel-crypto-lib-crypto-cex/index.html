    @cexiolabs/сrypto.cex

    CEX Crytpo Wallet - Реализации контрактного интерфейса CrytpoWallet через WTF2/pss-provider-wtf2

    Configuration

    Этот проект содержит конфигурационную информацию по интеграции с CEX Crypto Wallet (WTF2 и pss-provider-wtf2). Включает в себя:

    • конфигурацию криптокошельков и хот-адресов
    • набор вспомогательных скриптов (генерация ключей PSSES, загрузка ключей в PSSES, etc.)
    • загрузка ключей в PSS ES через Pipeline

    Немного теории

    В криптовалюте каждому wallet(группа блокчейн адресов) есть соответствующий приватный ключ, с помощью которого нужно подписывать(ЭЦП) транзакции перед отправкой в блокчейн. Процессниг-сервисам не нужно знать деталей работы с конкретным блокчейном, и процесс работы делегирован вспомогательным сервисам:

    • wtf2 - фасад в блокчейн
    • pss-es - Экспертная система, которая выполняет подпись транзакций (должна знать приватный ключ от wallet)
    • pss-provider-wtf2 - процессинг депозитов, виздавалов, генерация адресов. Этот сервис скрывается под собой работу с wtf2 и pss-es

    Установка ключей в PSS ES

    Одна из задач системы PSS ES - проводить подпись транзакций для блокчейнов. Для осуществления данной операции, PSS ES должна знать приватный ключ кошелька. PSS ES не хранит ключи в своей конфигурации (а лишь в памяти процесса) и следовательно, теряет их после перезапуска. Ключи от кошельков передаются в PSS ES средствами предоставляемого PSS ES API. Конфигурация данного сервиса хранит лишь секретную фразу, которая была использована для шифрования приватного ключа клиентом (нами).

    На момент написания этого документа, файл конфигурации PSS ES находится в d1dev-test:/app/conf/psses/dev/config-processing.json. После редактирования файла требуется помощь коллег из PSS Team, для перезагрузки PSS ES.

    Повторимся, PSS ES должна знать приватные ключи от wallet-ов, чтобы подписывать исходящие транзакции (виздравалы)

    Для автоматизации установки ключей, имеем следующее:

    • конфиг-файл ./keys-upload-config.json - содержит список зашифрованных ключей (массив keys)
    • скрипт ./bin/keys-upload.js - программа отправляющая содержимое конфига в PSS ES
    • скрипт ./bin/encode_ccw_key.js - программа шифрования приватного ключа (в PSS ES ключи передаются в зашифрованном виде)

    Выполнение keys-upload.js настроено в GitLab CI (Pipeline)

    keys-upload-config.json

    Мерчанту PSS (т.е. нам CryptoPay), по каждой валюте, создается 1+ элемент(ов) в массиве keys (для каждого приватного ключа который участвует в подписи транзакций):

    {
    "mid": "cpwtf", 
    "currency": "BTC", 
    "es": "psse",
    "key": "76b72284db5a5ddf361cde01113ce3098a4e292ccb2411385bf6859d53984fcf4f27ffac011de67aae604c8745d89ecbb17020072542f09eea6536879309f4f7b5f587e9be53c81766956bfb1bb969b8afabfd5ae6182ae16154cbf12c7eeb6026acf1f542fe9242c5ea1957ce2a549c"
}

    Где:

    • mid - имя мерчанта в PSS. Мы, CryptoPay, имеем два имени cpwtf и cpwtf-demo для evolution и presentation окружений соответственно
    • currency - идентификатор крипто-валюты
    • es - имя экспертной системы, куда устанавливается ключ
    • key - приватный ключ крипто-кошелька зашифрованный секретной фразой с помощью aes-128-cbc

    keys-upload.js

    $ ./bin/keys-upload.js

    encode_ccw_key.js

    $ ./bin/encode_ccw_key.js <keychains/privateKey> [PSSES shared secret]

    Note: Если параметр PSSES shared secret упустить, скрипт сгенерирует пароль самостоятельно

    • Результатирующий ключ добавляется в keys-upload-config.json.
    • Пароль PSSES shared secret добавился в конфигурацию PSS ES (при помощи коллег из PSS Team)

    Генерация нового wallet

    1. Get list of supported currencies: GET /currencies. See swagger: https://pss-provider-wtf2.dev.kube/api/#/currencies/CurrencyController_getCurrencies 
      curl -X GET "https://pss-provider-wtf2.dev.kube/currencies" -H "accept: application/json"
    2. Create new wallets for supported currencies: POST /wallets. See swagger: https://pss-provider-wtf2.dev.kube/api/#/wallets/WalletController_generateWallet 
      {
    "currency": "ATOM",
    "keychainOwners": [
        "psses"
    ],
    "name": "cryptopay-evolution-atom",
    "processingUrl": "https://evolution-cryptopay.dev.kube/v2/callback/cryptoproviders/cex/wtf2pss/ATOM/c8f93cc5-4c6f-44b7-9db4-bf089a7fc7f7",
    "encrypt": true
}
    3. If "multiSign": true - use this body: 
      {
    "currency": "ONT",
    "keychainOwners": [
        "psses",
        "faes"
    ],
    "name": "cryptopay-evolution-ont",
    "processingUrl": "https://evolution-cryptopay.dev.kube/v2/callback/cryptoproviders/cex/wtf2pss/ONT/04398c66-94e3-4b66-9c97-ab45d9b1d446",
    "encrypt": true
}
